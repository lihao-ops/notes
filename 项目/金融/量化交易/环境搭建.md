# 环境搭建



## 一、MySQL8.0

### 1.方案

> 在本机Windows电脑上搭建环境开机自启动。

原因：

1. 消耗：云服务器部署搭建首先会需要消耗很大的磁盘空间。
2. 丢失：且服务器到期后未续费存在数据丢失的风险。
3. 资源：本机资源充足，且无需额外费用。
4. 方便：数据读取无网络传输消耗，可直接本机读取。

### 5.my.ini

>在安装目录根目录下新建一个my.ini文件用于指定个人配置

```ini
[mysqld]
# 设置3306端口
port=3306
# 设置mysql的安装目录
basedir=D:\\MySQL8.0
# 设置mysql数据库的数据的存放目录
datadir=E:\\dataStore\\MySQL
# 允许最大连接数
max_connections=200
# 允许连接失败的次数。
max_connect_errors=10
# 服务端使用的字符集默认为utf8mb4
character-set-server=utf8mb4
# 创建新表时将使用的默认存储引擎
default-storage-engine=INNODB
# 默认使用“mysql_native_password”插件认证
#mysql_native_password
default_authentication_plugin=mysql_native_password
[mysql]
# 设置mysql客户端默认字符集
default-character-set=utf8mb4
[client]
# 设置mysql客户端连接服务端时默认使用的端口
port=3306
default-character-set=utf8mb4
```

### 7.进入bin

>从此cmd进入到安装目录的bin\目录下

![image-20250525223509341](https://notes-1307435281.cos.ap-shanghai.myqcloud.com/note/master/202505252235391.png)

注意：刚开始没有生效的原因是：还没有指定哪个磁盘，故此无法进行指定目录的切换！，这里先指定的D:

```bash
#我本地安装MySQL的目录所在磁盘
D:
```

>随后进入安装目录下的bin

```bash
cd D:\MySQL8.0\bin
```





### 8.指定自定义my.ini安装

>由于我本次自定义的my.ini文件所在地址是D:\MySQL8.0\my.ini,故此指定安装，否则将要读取在C盘默认的文件夹自动生成的.ini/.conf文件

```bash
D:\MySQL8.0\bin>D:\MySQL8.0\bin\mysqld.exe --install MySQL --defaults-file="D:\MySQL8.0\my.ini"
```

>继续指定安装

```bash
D:\MySQL8.0\bin>D:\MySQL8.0\bin\mysqld.exe --defaults-file="D:\MySQL8.0\my.ini" --initialize --console
```

### 9.启动MySQL

>启动刚刚自定义名称的MySQL

```bash
D:\MySQL8.0\bin>net start MySQL80
```

````bash

MySQL80 服务已经启动成功。
````

### 11.修改密码

>进入后修改密码

```bash
mysql> ALTER USER 'root'@'localhost' IDENTIFIED BY 'Q836184425';
Query OK, 0 rows affected (0.01 sec)
```

注：其中Q836184425为指定的root自定义密码。

>退出或者刷新使新密码生效

```bash
mysql> EXIT
Bye
```

>使用新密码登录

```bash
D:\MySQL8.0\bin>mysql -u root -p"Q836184425"
```

### 13.net命令必须以管理员身份运行

>注意

由于安装的时候也是用的管理员身份。所以在执行net等系统命令时还是需要用管理员身份运行cmd.exe才行

但mysql -u等日常命令并不需要。且此时MySQL8.0应该已是开机自启动状态！
