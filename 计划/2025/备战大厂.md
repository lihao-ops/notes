>事务调优（保证性能 + 避免死锁）

这一块特别重要：

- 避免长事务
- 避免大批量 UPDATE/DELETE
- 避免隐式事务
- 隔离级别选择（RR / RC）
- 控制事务内 SQL 顺序
- 避免过度 SELECT FOR UPDATE
- 使用合适索引缩小锁范围



>实战案例

| 场景                   | 类型                | 价值         |
| ---------------------- | ------------------- | ------------ |
| 转账操作死锁           | 行锁                | 你已经掌握   |
| 订单扣库存死锁         | Next-Key Lock       | 大厂真实事故 |
| 秒杀场景死锁           | 插入意向锁          | 高并发套路   |
| 唯一索引冲突导致死锁   | 二级索引 → 主键回表 | 业务里常发生 |
| 乐观锁与悲观锁混用死锁 | 错误加锁策略        | 高级知识点   |
